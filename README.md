# android-codestudio

A Claude skill for generating Android applications fully compatible with **Code Studio** (mobile Android IDE).

---

## What it does

This skill ensures that every Android project generated by Claude respects the strict constraints of Code Studio — without the user having to specify them each time.

Code Studio is a mobile Android IDE with its own compiler and project conventions that differ significantly from standard Android Studio. This skill encodes all those differences so Claude never generates incompatible code by accident.

---

## When to use it

Trigger this skill when the user:

- Asks to **create, build, or generate an Android app**
- Mentions **Code Studio** alongside Android development
- Requests **Java Android code** (activities, layouts, services, etc.)
- Asks to port or adapt an existing Android project to Code Studio

---

## Key constraints enforced

| Area | Rule |
|---|---|
| Build system | No Gradle, no Maven, no `build.gradle` |
| SDK | `minSdkVersion=21`, `targetSdkVersion=29` |
| Java | No lambdas, no streams, no method references, no `var` |
| Libraries | No AndroidX — use classic `android.support.*` ≤ 28.0.0 |
| Language | Java preferred; Kotlin only if explicitly requested |
| Layout file | `layout_main.xml` (not `activity_main.xml`) |
| Values file | Single `values.xml` (not separate `colors.xml`/`styles.xml`) |
| Manifest | At project root, icon via `@drawable/ic_launcher` |

```
ApplicationName/
├── assets/
├── libs/

│   ├── drawable/
│   ├── drawable-hdpi/
│   ├── drawable-ldpi/

│   ├── drawable-xhdpi/
│   ├── drawable-xxhdpi/
│   ├── layout/
│   │   └── layout_main.xml      ← Code Studio naming convention
│   ├── values/
│   │   ├── strings.xml
│   │   └── values.xml           ← colors + styles merged here
│   └── xml/
└── src/
    └── com/example/appname/
        └── MainActivity.java
```

## Limitations

- Kotlin is supported but not the default — only use when explicitly requested
- No Jetpack components (Compose, Navigation, WorkManager, Room, LiveData, ViewModel)
- RecyclerView is possible via `android.support.v7.widget` but ListView is preferred
- Image loading libraries: only old Glide 3.x `.jar` or manual `BitmapFactory` + `HttpURLConnection`
